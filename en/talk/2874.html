SnowOne 2020 (29.02.2020 — 29.02.2020)

ByteBuffers and beyond: Working with foreign memory in Java

Vladimir Ivanov

Direct buffers are, to date, the only supported way to access foreign, off-heap memory. Despite their success, direct buffers suffer from significant limitations --- stateful-ness, lack of addressing space, non-deterministic deallocation to name a few --- which makes them a less-than-ideal choice under certain workloads. In this talk we paint the path to the future: a safe, supported, and efficient foreign memory access API for Java. By providing a more targeted solution to the problem of accessing foreign memory developers will be free of those limitations but, in addition, they will also enjoy improved performance. The new API is designed from the ground-up with JIT optimizations in mind - and all without sacrificing memory access safety. The talk will cover Memory Access API (JEP 370) which is targeted to JDK 14 as an incubating module.
