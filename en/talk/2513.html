How to develop a successful Kubernetes native application using Quarkus

Alex Soto Bueno

Приходите на этот доклад, чтобы узнать, как разрабатывать успешные нативные Kubernetes-приложения, используя Quarkus — экосистему на базе Java для разработки нативных, облачных, контейнерных, безсерверных и оптимизированных под Kubernetes приложений.

Сегодня Kubernetes по сути стал нашей главной платформой по запуску приложений. И это изменение, конечно же, повлияло на то, как мы пишем наши приложения.

Если раньше мы сразу разрабатывали монолитное приложение, где все уже было готово к работе, то теперь мы разбиваем этот монолит на (микро) сервисную архитектуру, где все элементы взаимосвязаны через сеть.

И хоть это и может показаться простым на первый взгляд, на самом деле все не так просто, так как этот новый подход привносит ряд сложностей, ранее отсутствовавших в монолитной архитектуре.

Приходите на этот доклад, чтобы узнать, как разрабатывать успешные нативные Kubernetes-приложения, используя Quarkus — экосистему на базе Java для разработки нативных, облачных, контейнерных, безсерверных и оптимизированных под Kubernetes приложений.
