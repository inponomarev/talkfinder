DotNext 2020 Msc (02.12.2020 — 05.12.2020)

Workshop: Investigating memory leaks in .NET with WinDBG (part 2)

Mikhail Yarichuk

In this workshop, we will go through the most common memory problems and try to find out what is wrong with a few examples in practice. The devil is not so black as he is painted: research on memory dumps is laborious and time-consuming, but often irreplaceable.

People say that debugging code is like being the hero of a crime drama, where you are a killer and a detective at the same time. Although most often detectives investigate murders committed by others.

But there are no ideal crimes, the criminal always makes this or that mistake. The same is true when examining memory dumps for leaks and other problems - one way or another, you can always get to the bottom of it.

In this workshop, we will go through the most common memory problems and try to find out what is wrong with a few examples in practice. The devil is not so black as he is painted: research on memory dumps is laborious and time-consuming, but often irreplaceable.
