SPTDC 2020 (06.07.2020 — 09.07.2020)

Programming for persistent memory

Michael Scott

This class will explore a variety of topics in the rapidly evolving field of persistent memory, including hardware (memory and processor) foundations; formal correctness criteria (memory persistency, durable linearizability); persistent data structures; run-time systems for failure atomicity; dynamic memory allocation; and operating system support for failure recovery and for safe, user-level sharing.

<p>For the coming decade, dynamic random-access memory (DRAM) is likely to be replaced in many applications by new forms of non-volatile memory (NVM), which retain their contents when power is lost. This prospect raises the possibility that certain data formerly kept in files might instead reside "in memory" across program runs and even system crashes, leading to significant reductions in program complexity and run-time overhead. To achieve this vision, this class will explore a variety of topics in the rapidly evolving field of persistent memory, including hardware (memory and processor) foundations; formal correctness criteria (memory persistency, durable linearizability); persistent data structures; run-time systems for failure atomicity; dynamic memory allocation; and operating system support for failure recovery and for safe, user-level sharing.</p> <p><a href="https://www.dropbox.com/s/9p4biqo49vx8o1y/NVM_References.docx?dl=0">References.</a></p>
