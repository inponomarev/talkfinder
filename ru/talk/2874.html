Работа с памятью вне Java-кучи: есть ли будущее у ByteBuffer'ов?

Владимир Иванов

На сегодняшний день, ByteBuffer’ы - часть NIO (Non-blocking I/O) API - являются единственным поддерживаемым способом доступа к памяти вне Java-кучи (off-heap). Несмотря на популярность, API страдает от существенных ограничений, что не позволяет достичь оптимальной производительности в некоторых сценариях использования.

Доклад будет посвящен Memory Access API, который должен появиться в JDK 14 в виде инкубационного модуля (JEP 370). Новый API предоставляет безопасный и эффективный доступ к внешней памяти из Java: дизайн "с нуля" освобождает от ограничений присущих NIO и позволяет достичь более высокой скорости работы.
