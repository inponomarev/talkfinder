Оборачиваем существующее REST API в GraphQL (методология за 1000000 рублей)

Павел Черторогов

Вам нужен GraphQL, но ваш бэкенд уже лет как 10 написан на REST API, и переписать его просто нереально? Не беда! Вы можете обернуть его в GraphQL. GraphQL как бы станет удобной проксей для доступа к вашему «старому» API.

<p>А реально обернуть REST API в GraphQL? И как сделать R&D для этой технологии?

Обычно обкатку технологии принято делать на TodoMVC. Так и поступим.</p> <p>А коль Павел человек непростой, то придется брать и непростое Todo-приложение, к примеру, систему управления проектами Wrike.</p> <p>Возьмем <a href="https://developers.wrike.com/api/v4">REST API</a> от Wrike:</p> <ul> <li>22 entity;</li> <li>87 методов.</li> </ul> <p>И потратим 160 часов на:</p> <ul> <li>134 GraphQL-типа;</li> <li>1002 поля и аргумента;</li> <li>51 явную прямую связь между entity по id;</li> <li>26 неявных обратных связей через фильтры.</li> </ul> <p>Весь полученный опыт Павел сожмет в часовой доклад. Он поделится наработками, набитыми шишками, идеями и мыслями о том, как хорошо и качественно обернуть REST API в GraphQL. Приходите посмотреть, будет интересно!</p> <p>А перед докладом можно посмотреть на то, что получилось, и о чем спикер будет рассказывать:</p> <ul> <li>Живой <a href="https://graphql-wrike.herokuapp.com/">GraphQL-прокси</a>;</li> <li><a href="https://graphql-wrike.herokuapp.com/voyager">Диаграмма связей</a> настоящего «волосатого» GraphQL;</li> <li><a href="https://github.com/nodkz/wrike-graphql">Репозитарий</a> с кодом.</li> </ul>
